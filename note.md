# 背景

自动驾驶决策即在道路交通环境中为自车安排一系列动作/策略序列。典型动作可分为宏观驾驶意图（直行、变道、减速/停车、避让等）与微观连续控制（油门、刹车、转向）。本研究采用场景图建模（GNN）+ 强化学习（PPO）：先将感知输出构造成交通场景图，经GNN提取全局结构语义，再由层次化策略（高层意图 + 低层控制）输出可执行动作，以在扰动/异常感知下保持鲁棒。

# 约束

车辆（自车）的约束如下表所示：

| 编号 | 类别         | 名称                         | 说明                                                       |
|------|--------------|------------------------------|------------------------------------------------------------|
| 1    | –            | 车辆标识                     | 哪一辆车（自车实例/车辆配置）                              |
| 2    | 交通规则     | 速度上限/下限                | 受道路限速与车辆最低可控速度约束                           |
|      |              | 信号灯/路口通行规则          | 红灯停车、路口让行、斑马线礼让                             |
|      |              | 车道边界/线型                | 实线不可变道、路肩不可驶入                                 |
| 3    | 安全时距     | 纵向时距/最小TTC             | 与前车/横向来车保持安全余度                                |
| 4    | 动力学约束   | 最大加速度/减速度、最大转向角速度、最大横摆率 | 车辆动力学与舒适性（jerk）限制                     |
| 5    | 时段滑动约束 | 变道频次/急制动频次（滑动窗口） | 任意Δt内不超过阈值，限制频繁激进操作                       |
| 6    | 资源/健康    | 电量/温度（可选）            | 续航或热约束（如仿真可忽略或简化为惩罚项）                 |

# 输入

- `route_waypoints`：路线/目标点（等价于“需求”）
- `scene_detections`：感知输出（车辆/行人/骑行者/障碍物、车道、交通灯）
- `map & rules`：HD-Map片段（限速、车道拓扑、优先权）
- `vehicle_state`：自车状态（位置、速度、姿态、控制量）与车辆参数（动力学上下限）

# 约束分析

## 目标

构建驾驶规则检查模块，接收约束列表初始化，提供：

- `detect(action)`：给定候选动作，判断是否满足规则/安全约束
- `arrange(action)`：应用该动作更新状态/轨迹缓存，返回是否成功

## 关键字段（用于约束校验）

- **候选动作**：意图（直行/变道/停车…）、控制量（a, δ），作用时窗[t_s, t_e]
- **环境**：前车/侧后车相对距离、相对速度、信号灯状态、限速、可变道标志
- **安全**：TTC、最小间隙、可达性；动力学上限（|a|、|jerk|、|yaw_rate|）

## 怎么满足约束

- **交通规则**：基于HD-Map与信号灯状态做动作掩码
- **时间间隔/冲突**：用短时可达性/碰撞预测替代区间树；若预测与他车轨迹重叠则判冲突
- **滑动窗口**：统计近Δt内变道/急制动次数，超过阈值则屏蔽相应激进行为
- **安全时距**：基于IDM/TTC阈值对跟驰/并线动作做掩码或惩罚

# 智能体决定做什么

- **高层（意图层）**：{保持车道、左/右变道、减速/停车、超越/让行}
- **低层（控制层）**：在意图指令下输出连续控制（油门/刹车/转向）

**执行顺序**：优先满足行进任务，在规则与安全约束下择优执行意图；必要时触发让行/停车

# 图神经网络的处理

- **无GNN**：拼接扁平状态向量 → PPO
- **有GNN**：场景图→GNN→全局向量→PPO

## 节点

自车、周车、行人/骑行者、交通灯、车道段/路口

## 边

相对位置/速度、可变道关系、优先权/让行关系、可达连边

## 池化

读出全局场景向量

# 设计

## 需要哪些东西

- 路线/目标点
- 动态目标集合
- 车道/路口拓扑与信号灯
- 自车状态与车辆参数
- 进度状态
- 规则检查缓存

## 图神经网络设计

- **做什么**：把非定长、多主体的交通场景转为定长语义向量，弱化单点误检影响
- **输入/输出**：场景图 → 节点嵌入 → 全局池化 → [1×D]环境向量
- **提取什么**：全局可达性/冲突风险、通行优先、可变道性、信号约束、主导交互体

## 强化学习网络设计

- **最高层**
  - 动作空间：{直行、左变、右变、减速/停车、超越/让行}
  - 输出：[K×1] 概率；动作掩码按约束裁剪（红灯禁行、不可变道等）

- **第二层**
  - 条件于意图，输出连续控制（PPO/或SAC）；代价包含安全/舒适/规则惩罚

## 怎么构图

- 自车节点 ↔ 道路/车道段节点
- 自车节点 ↔ 周车节点
- 自车节点 ↔ 信号灯/路口节点
- 车道段之间有拓扑边

**层级视角**：路网/路口拓扑 → 车道占用/可达 → 车与车/人交互 → 自车决策

## 图节点怎么编码

- **自车节点**：位置/姿态、速度向量、加速度/横摆率、控制输入、车身尺寸
- **周车节点**：相对位置、相对速度、距离/时距、尺寸、置信度
- **行人/骑行者节点**：位置/速度/过街概率（可简化为相对位姿与速度）
- **信号灯节点**：{dist, one-hot(红/黄/绿/未知)}
- **车道/路口节点**：可变道标志、限速、曲率/航向差、是否路口

## 维度怎么变化

1. 原始环境
2. 构图（节点数N、边数E均不定）
3. 编码：每类节点MLP→统一维度d
4. GNN传播与全局池化：N×d → 1×D
5. Actor（意图）：1×D → K（掩码后采样）
6. Critic：1×D → V；控制层接收1×(D+意图one-hot)输出连续控制

## 停止条件（episode终止）

1. 到达目标/终点
2. 碰撞/严重违章
3. 超时
4. 车辆静止且无可行动作

# 实验与扰动对比

- **无GNN基线**：扁平拼接（ego/IMU/GNSS/最近若干车/信号灯/车道）→ PPO
- **有GNN**：场景图→GNN→PPO
- **扰动**：加噪（位置/速度）、漏检/误检、信号灯误读、随机延迟；比较任务完成率、碰撞率、红灯违章、均方横向偏差、急制动/变道频次

**预期**：有GNN在扰动下更稳
